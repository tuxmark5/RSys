Duomenų bazės paruošimas Ubuntu operacinėje sistemoje

Šiame skyrelyje bus aptarta, kaip reikia paruošti PostgreSQL DBVS taip, kad ši būtų tinkama nuotoliniam prisijungimui. Tai atlikti yra visiškai nebūtina, jei pakanka vietinės DB funkcionalumo.

Norint paruošti serverį, tinkamą notoliniam prisijungimui, reikia:

  1. Instaliuoti PostgreSQL 9.1 (arba naujesnę) DBVS:
    sudo apt-get install postgresql-9.1
  2. Prisijungti naudotoju postgres:
    sudo su postgres
  3. Inicijuoti duomenų bazę:
    psql < init-postgres-admin.sql
    Šis žingsnis sukurs naują duomenų bazę pavadinimu db ir naują naudotoją admin su slaptažodžiu pass1234, kuris turės pilnas teises prie sukurtos duomenų bazės. Prieš atliekant šį žingsnį, rekomenduojama pakeisti failą init-postgres.sql taip, kad naujos duomenų bazės ir administratoriaus naudotojo vardai bei slaptažodis atitiktų poreikius.
  4. Atsijungti nuo naudotojo postgres:
    exit
  5. Sukurti reikiamas lenteles ir struktūras naujoje db:
    psql -d db -U admin -h 127.0.0.1 -f init-postgres.sql

Atlikus šiuos žingsnius, bus galima prisijungi prie DB naudojant RSys sistemą. Prisijungus rekomenduojama importuoti pradinius duomenis ir sukurti reikiamų naudotojų paskyras. Plačiau apie tai žr. skyrelius "Duomenų importavimas" ir "Naudotojų administravimas".

Duomenų bazės paruošimas kitose operacinėse sistemose

Duomenų bazės paruošimas kitose operacinėse sistemose yra analogiškas aprašytam, tik gali skirtis pačios DBVS instaliavimo procedūra ir būdas, kuriuo duomenų bazei yra pateikiami pradiniai .sql failai.
